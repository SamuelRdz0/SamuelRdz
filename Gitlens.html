Funcionalidad de GitLens

GitLens es una extensión para Visual Studio Code que mejora las capacidades de Git integradas en el editor. A continuación, se describe su funcionalidad mediante su uso:

1. Blame Annotation

GitLens muestra detalles de quién y cuándo se modificó una línea específica de código. Al colocar el cursor sobre una línea, despliega información del autor, fecha del commit y un mensaje corto.

2. Exploración del Historial

Permite ver el historial de cambios de un archivo o de una línea específica, facilitando analizar la evolución del código.

3. Comparación de Commits

Facilita comparar un commit anterior con el estado actual del archivo o entre dos commits. Esto ayuda a visualizar diferencias y entender qué cambios se han hecho.

4. Visualización del Historial del Repositorio

Ofrece una vista detallada del historial del repositorio con commits, ramas y etiquetas, proporcionando una visión clara del progreso del proyecto.

5. Explorador de Repositorios

Permite navegar por las diferentes ramas y etiquetas del proyecto, mostrando los cambios realizados en cada una.

6. Acciones de Commit

Facilita realizar commits, deshacerlos y ver el contenido de cada uno sin necesidad de usar la línea de comandos.

7. Integración con Pull Requests

Se integra con GitHub, Bitbucket y GitLab, mostrando detalles de los pull requests, lo que ayuda a revisar los cambios antes de integrarlos.

En resumen, GitLens mejora la comprensión del historial de cambios y facilita la navegación, comparación y análisis de versiones del código desde el editor, mejorando la productividad en el uso de Git.
